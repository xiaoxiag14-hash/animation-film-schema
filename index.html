Animation Film Metadata Schema

A metadata schema for describing, managing, searching, and reusing animation film information.

1. Opening Narrative

본 프로젝트는 애니메이션 영화를 체계적으로 설명하고 관리하기 위한 메타데이터 스키마를 설계하는 것을 목적으로 한다. 애니메이션 영화는 단순히 제목, 감독, 개봉일과 같은 기본 정보만으로 설명되기 어렵다. 작품마다 장르, 제작 국가, 상영 시간, 언어, 애니메이션 스타일, 주제, 제작사, 음악, 캐릭터 디자인 등 다양한 정보가 함께 작용하기 때문이다. 따라서 본 스키마는 애니메이션 영화를 검색, 비교, 분석, 관리할 수 있도록 기본 기술 요소와 애니메이션 도메인 특화 요소를 함께 포함하였다.

이 스키마의 주요 이용자는 일반 시청자, 영화 및 애니메이션 연구자, 콘텐츠 관리자, 그리고 산업 종사자이다. 일반 시청자는 장르, 언어, 주제, 애니메이션 스타일 등을 통해 원하는 작품을 쉽게 찾고 비교할 수 있다. 연구자는 여러 작품의 제작 경향, 장르적 특징, 주제 변화, 애니메이션 기법의 차이를 분석하는 데 이 데이터를 활용할 수 있다. 관리자는 영화 자료를 일관된 형식으로 정리하고, 향후 다른 시스템이나 데이터베이스와 연계할 수 있다.

초기 설계에서는 영화의 기본 정보 중심으로 스키마를 구성하였다. 그러나 피드백을 반영하여 최종 스키마에서는 기존 OTT 플랫폼의 단순 검색 정보와 차별화될 수 있도록 애니메이션 스타일, 주제, 음악 감독, 캐릭터 디자이너, 시리즈 관계와 같은 요소를 추가하였다. 특히 OTT 플랫폼이 이용자 맞춤형 추천에 초점을 두는 반면, 본 스키마는 다양한 이용자가 애니메이션 영화 정보를 탐색, 분석, 재사용할 수 있도록 지원하는 데 목적이 있다.

이 스키마는 Dublin Core Terms, Schema.org, EBUCore와 같은 기존 메타데이터 표준을 참고하여 설계되었다. 이를 통해 기본적인 상호운용성을 확보하고, 동시에 애니메이션 영화의 특성을 반영하는 자체 요소를 추가하였다. 따라서 본 스키마는 표준성과 도메인 특수성을 함께 고려한 소규모 애니메이션 영화 메타데이터 모델이라고 할 수 있다.

Schema Name Prefix Namespace Reason for Use

Schema.org schema https://schema.org/ Movie, CreativeWork와 관련된 감독, 장르, 상영 시간, 언어, 제작사, 시리즈 관계 등의 요소를 참고하기 위해 사용하였다.

Animation Metadata Schema anim http://example.org/animation-schema/ 기존 표준만으로 충분히 표현하기 어려운 애니메이션 스타일, 주제, 캐릭터 디자이너 등의 도메인 특화 요소를 정의하기 위해 자체 네임스페이스로 사용하였다.

Draft 2에서는 Media Resource Ontology도 검토하였으나, 최종 스키마에서는 전체 영화 단위의 기술을 중심으로 설계하였기 때문에 미디어 조각, 트랙, 프레임 단위의 세부 기술은 주요 요소로 포함하지 않았다. 대신 EBUCore와 Schema.org를 중심으로 미디어 자원의 기본 정보와 애니메이션 도메인 특화 정보를 함께 표현하였다.

3.2 Data Dictionary

title

Definition: Official title of the animated film. Required: Yes. Repeatable: No. Data type: String. Example: Your Name.

director

Definition: Person or persons who directed the animated film. Required: Yes. Repeatable: Yes. Data type: String. Example: Makoto Shinkai.

date

Definition: Release year of the film. Required: Yes. Repeatable: No. Data type: xs:gYear. Example: 2016.

genre

Definition: Genre category of the animated film. Required: Yes. Repeatable: Yes. Controlled vocabulary: Fantasy, Adventure, Romance.

description

Definition: Short summary of the film's story, background, or content. Required: No. Repeatable: No. Data type: Text.

animationStyle

Definition: Visual or production style of the animation. Required: Yes. Repeatable: Yes. Controlled vocabulary: Anime, 2D Animation, 3D Animation, Stop Motion, Claymation, Cut-out Animation.

duration

Definition: Running time of the film. Required: Yes. Repeatable: No. Data type: String, preferably ISO 8601 duration format. Example: PT106M.

inLanguage

Definition: Main language or languages used in the film. Required: Yes. Repeatable: Yes. Data type: String. Example: Japanese.

theme

Definition: Main thematic focus of the film. Required: No. Repeatable: Yes. Controlled vocabulary: Identity, Time Travel, Friendship, Love, Courage, Family, Growth, Social Issues.

productionCompany

Definition: Company or studio responsible for producing the film. Required: Yes. Repeatable: Yes. Data type: String. Example: Studio Ghibli.

musicDirector

Definition: Person or group responsible for the film music or soundtrack. Required: No. Repeatable: Yes. Data type: String. Example: Joe Hisaishi.

characterDesigner

Definition: Person responsible for the character design of the animation. Required: No. Repeatable: Yes. Data type: String. Example: Masayoshi Tanaka.

isPartOf

Definition: Larger series, franchise, or related work that the film belongs to. Required: No. Repeatable: No. Data type: String. Example: Spider-Verse Series.

4. XSD File

The XSD file is hosted as part of this website. Sample XML records can reference this XSD file for validation.

Open XSD File

<?xml version="1.0" encoding="UTF-8"?>

<xs:schema
    xmlns:xs="http://www.w3.org/2001/XMLSchema"
    targetNamespace="http://example.org/animation-schema"
    xmlns="http://example.org/animation-schema"
    elementFormDefault="qualified">

    <xs:annotation>
        <xs:documentation>
            Animation Film Metadata Schema.
            This schema is designed to describe animated films for discovery,
            management, comparison, and research use. It combines general film
            metadata elements with animation-specific elements such as animation
            style, theme, music director, character designer, and series relation.
        </xs:documentation>
    </xs:annotation>

    <!-- Design intent: GenreType uses a controlled vocabulary to improve consistency in genre description. -->
    <xs:simpleType name="GenreType">
        <xs:annotation>
            <xs:documentation>
                Controlled vocabulary for the main genre category of an animated film.
            </xs:documentation>
        </xs:annotation>
        <xs:restriction base="xs:string">
            <xs:enumeration value="Fantasy"/>
            <xs:enumeration value="Adventure"/>
            <xs:enumeration value="Romance"/>
        </xs:restriction>
    </xs:simpleType>

    <!-- Design intent: AnimationStyleType highlights the animation-specific characteristics of the film. -->
    <xs:simpleType name="AnimationStyleType">
        <xs:annotation>
            <xs:documentation>
                Controlled vocabulary for the visual or production style of animation.
            </xs:documentation>
        </xs:annotation>
        <xs:restriction base="xs:string">
            <xs:enumeration value="Anime"/>
            <xs:enumeration value="2D Animation"/>
            <xs:enumeration value="3D Animation"/>
            <xs:enumeration value="Stop Motion"/>
            <xs:enumeration value="Claymation"/>
            <xs:enumeration value="Cut-out Animation"/>
        </xs:restriction>
    </xs:simpleType>

    <!-- Design intent: ThemeType supports thematic searching and research use. -->
    <xs:simpleType name="ThemeType">
        <xs:annotation>
            <xs:documentation>
                Controlled vocabulary for the main themes represented in the film.
            </xs:documentation>
        </xs:annotation>
        <xs:restriction base="xs:string">
            <xs:enumeration value="Identity"/>
            <xs:enumeration value="Time Travel"/>
            <xs:enumeration value="Friendship"/>
            <xs:enumeration value="Love"/>
            <xs:enumeration value="Courage"/>
            <xs:enumeration value="Family"/>
            <xs:enumeration value="Growth"/>
            <xs:enumeration value="Social Issues"/>
        </xs:restriction>
    </xs:simpleType>

    <xs:element name="animationFilm">
        <xs:annotation>
            <xs:documentation>
                Root element for one animated film metadata record.
                Each record describes one animated film as a complete work.
            </xs:documentation>
        </xs:annotation>

        <xs:complexType>
            <xs:sequence>

                <!-- Required because title is the primary access point for identifying the film. -->
                <xs:element name="title" type="xs:string">
                    <xs:annotation>
                        <xs:documentation>Official title of the animated film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Repeatable because some films have more than one director. -->
                <xs:element name="director" type="xs:string" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Director of the animated film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="date" type="xs:gYear">
                    <xs:annotation>
                        <xs:documentation>Release year of the animated film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Repeatable because a film can belong to more than one genre. -->
                <xs:element name="genre" type="GenreType" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Genre category of the animated film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="description" type="xs:string" minOccurs="0">
                    <xs:annotation>
                        <xs:documentation>Short summary or description of the film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Animation-specific element added to distinguish this schema from general movie schemas. -->
                <xs:element name="animationStyle" type="AnimationStyleType" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Visual or production style of animation.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="duration" type="xs:string">
                    <xs:annotation>
                        <xs:documentation>Running time of the film, preferably written in ISO 8601 duration format such as PT106M.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="inLanguage" type="xs:string" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Main language or languages used in the film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="theme" type="ThemeType" minOccurs="0" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Main theme or themes of the animated film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <xs:element name="productionCompany" type="xs:string" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Company or studio responsible for producing the film.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Added after feedback to represent the importance of music in animated films. -->
                <xs:element name="musicDirector" type="xs:string" minOccurs="0" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Person or group responsible for the film music or soundtrack.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Added after feedback to represent animation production roles more clearly. -->
                <xs:element name="characterDesigner" type="xs:string" minOccurs="0" maxOccurs="unbounded">
                    <xs:annotation>
                        <xs:documentation>Person responsible for character design.</xs:documentation>
                    </xs:annotation>
                </xs:element>

                <!-- Optional relation element for sequels, series, franchises, or adapted works. -->
                <xs:element name="isPartOf" type="xs:string" minOccurs="0">
                    <xs:annotation>
                        <xs:documentation>Series, franchise, or larger work that the film belongs to.</xs:documentation>
                    </xs:annotation>
                </xs:element>

            </xs:sequence>
        </xs:complexType>
    </xs:element>

</xs:schema>

The following sample XML records describe individual animated films and validate against the XSD file above. After publishing the website, the xsi:schemaLocation value can be replaced with the direct public URL of the hosted XSD file.

Sample Record 1: Your Name.

Open XML File

<animationFilm
    xmlns="http://example.org/animation-schema"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://example.org/animation-schema animation.xsd">

    <title>Your Name.</title>
    <director>Makoto Shinkai</director>
    <date>2016</date>
